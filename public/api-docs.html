🐙 OctoSHOT WhatsApp API

Complete documentation for the WhatsApp messaging platform API

📋 Overview

The OctoSHOT WhatsApp API provides a powerful interface for sending messages through WhatsApp. It supports single and bulk messaging with smart batching, spintax for message variation, and real-time delivery tracking.

Base URL

http://localhost:3002/api/v1

Key Features

📤 Multiple Message Types Text, Image, Document, Audio, Video, Location, Contact
📦 Bulk Messaging Send to multiple recipients with smart batching
🔄 Spintax Support Automatic message variation for uniqueness
⚡ Trust-Based Rate Limiting Dynamic limits based on account trust level
📊 Real-Time Status Tracking Sent → Delivered → Read status updates

🔐 Authentication

All API requests require two authentication headers:

Header Description Example
X-Device-UUID Your device's unique identifier 11666fa0-2cbb-4b1f-...
X-API-Token Your API authentication token 1e21fac6d84fa4ecb46...
💡
Where to find your credentials?

Go to WhatsApp Profiles and click on your device to view the UUID and API Token.

Example Request

POST /api/v1/messages/send-text // Headers: X-Device-UUID: 11666fa0-2cbb-4b1f-b6ba-29b63fd7ed51 X-API-Token: 1e21fac6d84fa4ecb46e8c32d0d0f0fc95e05a88 Content-Type: application/json

POST Send Text Message

Send a text message to one or multiple recipients. Supports bulk messaging and spintax.

POST /api/v1/messages/send-text

Request Body

Parameter Type Required Description
to string Required Phone number(s). Comma-separated for bulk.
message string Required Message content. Supports spintax.

Single Message Example

{ "to": "201234567890", "message": "Hello! This is a test message." }

Bulk Message Example

{ "to": "201234567890,201098765432,201111222333", "message": "{Hello|Hi|Hey}! Your order is {ready|prepared}." }

Response (Single)

{ "success": true, "messageId": "true_201234567890@c.us_3EB0...", "timestamp": 1765148664, "to": "201234567890@c.us" }

Response (Bulk)

{ "success": true, "mode": "bulk", "batchId": "ec84838a-6047-438f-8c29-...", "originalCount": 125, "duplicateCount": 20, "uniqueCount": 105, "blacklistedCount": 0, "recipientCount": 105, "batchCount": 3, "messagesPerBatch": 50, "intervalBetweenBatches": "12 hours", "trustLevel": 1, "estimatedCompletionTime": "25 hours", "message": "Messages queued. 20 duplicate numbers removed." }

📥 Incoming Messages Webhook

Receive real-time notifications when messages arrive on your WhatsApp device. Configure a webhook URL in your device profile settings.

Configuration

Go to WhatsApp Profiles → Edit Device → Set the رابط Webhook (optional) field.

Webhook Payload

The following JSON payload is sent to your رابط Webhook for each incoming message:

{
  "type": "incoming_message",
  "from": "201234567890",
  "from_name": "John Doe",
  "message_id": "false_201234567890@c.us_3EB0...",
  "body": "Hello, this is the message text",
  "type": "text",
  "timestamp": "2025-12-08T03:12:08.000+02:00",
  "is_group": false,
  "device_id": 1
}

Payload Fields

Field Type Description
type string Always "incoming_message"
from string Sender's phone number (without +)
from_name string Contact name if saved
message_id string WhatsApp message unique identifier
body string Message text content (empty for media)
timestamp string Message time in Cairo timezone (+02:00)
is_group boolean Whether message is from a group
device_id number Your device's internal ID

📭 Unread Message Notifier Webhook

Get notified when messages you sent remain unread (not delivered or delivered but not read) after a specified time period. You can configure up to 5 webhooks per device with different timers.

💡
Use Case

Perfect for follow-up automation - trigger a reminder sequence when customers don't read your messages.

Configuration

Go to WhatsApp Profiles → Edit Device → Webhook Notifications → Unread Messages tab.

Set timer value and unit (minutes/hours/days), then add your رابط Webhook.

Webhook Payload

{
  "type": "unread_message",
  "recipient": "201234567890",
  "message_id": "true_201234567890@c.us_3EB0...",
  "sent_at": "2025-12-08T03:12:08.000+02:00",
  "unread_duration_minutes": 5,
  "timer_setting": "5 minutes",
  "device_id": 1,
  "chat_history": [
    {
      "body": "Message text",
      "from_me": true,
      "timestamp": "2025-12-08T01:12:08.000Z",
      "local_timestamp": "2025-12-08T03:12:08.000+02:00",
      "type": "text"
    }
  ]
}

Payload Fields

Field Type Description
type string Always "unread_message"
recipient string Phone number message was sent to
message_id string WhatsApp message unique identifier
sent_at string When message was sent (Cairo timezone +02:00)
unread_duration_minutes number Configured timer in minutes
timer_setting string Human-readable timer (e.g., "5 minutes")
device_id number Your device's internal ID
chat_history array Last 10 messages in the conversation

👁️ Read No-Reply Notifier Webhook

Get notified when customers read your message but don't reply within a specified time. You can configure up to 5 webhooks per device.

⚠️
How it Works

The timer starts when the customer reads your message (blue ticks). If they don't reply before the timer expires, the webhook fires.

Configuration

Go to WhatsApp Profiles → Edit Device → Webhook Notifications → Read No-Reply tab.

Webhook Payload

{
  "type": "read_no_reply",
  "recipient": "201234567890",
  "message_id": "true_201234567890@c.us_3EB0...",
  "sent_at": "2025-12-08T03:12:08.000+02:00",
  "read_at": "2025-12-08T03:14:30.000+02:00",
  "no_reply_duration_minutes": 1,
  "timer_setting": "1 minutes",
  "device_id": 1,
  "chat_history": [...]
}

Payload Fields

Field Type Description
type string Always "read_no_reply"
recipient string Phone number message was sent to
sent_at string When message was sent (Cairo +02:00)
read_at string When customer read the message (Cairo +02:00)
no_reply_duration_minutes number Configured timer in minutes
chat_history array Last 10 messages in the conversation
💡
When Webhook Does NOT Fire

If the customer replies before the timer expires, the webhook will not be triggered.

📦 Bulk Messaging

Send messages to multiple recipients with smart batching to avoid WhatsApp rate limits.

How It Works

  1. Provide comma-separated phone numbers in the to field
  2. System detects bulk mode (2+ recipients)
  3. Duplicates are automatically removed
  4. Messages are split into batches based on your Trust Level
  5. Batches are scheduled with intervals between them
  6. Within each batch, messages are sent with 15-second delays
✅
Smart Features

Spintax is automatically resolved to create unique messages for each recipient.

🔄 Spintax Support

Spintax allows you to create message variations automatically. Use {option1|option2|option3} syntax.

Example

"message": "{Hello|Hi|Hey}! Your order is {ready|prepared|complete}."

Possible Outputs

  • "Hello! Your order is ready."
  • "Hi! Your order is prepared."
  • "Hey! Your order is complete."
  • ... and more combinations
💡
Why Use Spintax?

Unique messages for each recipient help avoid spam detection and make communications feel more personal.

📋 API Logs

Monitor all your API messages at /api-logs.

Features

  • View all sent messages with status
  • Filter by status or batch ID
  • Real-time status updates
  • Batch management (view and delete)
  • Configurable rows per page

📦 Batch Management

Manage your bulk message batches.

View All Batches

GET /api/logs/batches

Delete a Batch

DELETE /api/logs/batch/{batchId}

This removes all pending messages from the batch and cancels scheduled sends.

⚠️ Limitations

Limit Value Notes
Message delay 15 seconds Between each message in a batch
Batch interval (L1) 12 hours Between batches at Trust Level 1
Max total (L1) 300 messages Per campaign at Trust Level 1
Media size 16 MB WhatsApp limit for media files
Duplicate removal Automatic Duplicate numbers are filtered
